# Personalized print templates — drift status & de-fork backlog

The scheduler renders 32 personalized print templates
(`app/templates/{bowel_prep,egd,egd_phmii}/*-personalized.*.html`). Most are
hand-maintained **forks** of the canonical handout templates in the authoring
skills (vendored under `vendor/<skill>/templates/`). This doc records the drift
posture and the staged work to remove the forks.

## What guards drift today

`scripts/check_personalized_drift.py` + `scripts/personalized_provenance.json`
(wired into `make verify` via the meta repo, and the backend `make drift-check`).

It is a **provenance** gate: each personalized template records the sha256 of the
canonical it was last synced from. The gate FAILS when a canonical changes after
that baseline (i.e. NEW silent drift) or when a personalized template is
untracked. Re-baseline after reviewing + syncing a fork:

```
python3 scripts/check_personalized_drift.py            # gate
python3 scripts/check_personalized_drift.py --update   # accept current canonicals
```

**Scope boundary (important):** the gate catches drift introduced *after* the
baseline. It does **not** assert that a fork's current content already equals
`canonical + personalization` — several forks are known to be mildly stale today
(e.g. `suprep-standard-print` lost a `.meals .meal h3` selector a canonical CSS
tweak added). Removing that pre-existing staleness is the **de-fork** below, which
changes scheduled-patient PDF content and therefore needs per-template clinical
review — out of scope for the gate.

## Inventory

- **24 forked** (canonical exists, tracked by sha): the standard / infant /
  infant-enema / clenpiq-standard / suprep-standard / lactulose-standard /
  lactulose-infant families (colonoscopy + most combined), plus EGD and EGD+pH-MII.
- **8 backend-only "orphans"** (no skill canonical — reported as ORPHAN, not a
  failure): `combined-{suprep,clenpiq,lactulose}-standard-print-personalized.{en,es}`
  and `combined-lactulose-infant-print-personalized.{en,es}`.

### Orphans — reachable & intentional
The skill refuses to build `combined × {suprep,clenpiq,lactulose}` canonically
(the validate/render matrix boundary), but the scheduler **does** serve them:
`main.py` routes `procedure_type=combined` + `prep_type=suprep` →
`combined.render_pdf(... prep_type=suprep ...)` → the combined-suprep template,
and the `egdcolonsuprep{,86}` subdomains exist. So these are NOT dead files.
**Decision still open** (do not delete): either
1. **add the canonicals to the skill** (lift render.py's combined restriction for
   these preps — expands the Part-B variant matrix, needs clinical sign-off), or
2. **keep them backend-only** and make them honest — strip the misleading
   "GENERATED by build_personalized_templates.py — DO NOT HAND-EDIT" banner that
   6 of them currently carry (nothing regenerates them).

## Staged de-fork (= "Standardization Phase 2", A1 proper) — not yet done

Goal: make the 24 forks true derived artifacts of canonical (or eliminate the
committed forks entirely by personalizing at render time), so drift becomes
structurally impossible. Blocked on:
- The transforms differ per family (combined vs standard vs infant vs EGD) and the
  forks carry inconsistent strip behavior — a faithful generator must reproduce
  each before it can replace it.
- Regenerating changes scheduled-patient PDF content → needs per-template review +
  the existing PDF gates (`make verify-pdf-live`, calm + PDF/UA asserts).
- The orphan parent decision above.

Do it template-by-template behind the gate, re-baselining the manifest as each
fork is brought back in sync.
